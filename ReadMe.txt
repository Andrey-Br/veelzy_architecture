Слои
ui -> Bloc -> Service -> Repository

Через Слои не прыгаем. В Крайнем случае если экран маленький и у него нет Bloc, думаю ничего страшного если из ui сразу в слой Сервиса. 


Модели общие у всех кроме уровня Repository. И то только при необходимсти и они скрыты на этом слое.
Логика отдельной модели для уровня Repository в том, что с большой вероятностью модель которую используем мы в приложениии отличается от той, которую принимаем/отправляем c бэка.
И в этом случае отдельная модель репозитория служит как переходник, который мы можем "заточить" под конкретный JSON Сервера.
И из этого переходника получать полноценную модель, которую мы используем везде.

На остальных уровнях решил не делать дополнительные модели, так как скорее всего они будут тупо дублироваться, и реального смысла в этом не увидел.


Репозитории работают сами по себе и зависят только от интерфейса.

Сервис может содержать несклько репозиториев, которые передаются через конструктор при инициализации. Сам сервис зависит от своего интерфейса.

Bloc не имплеминтирует интерфейса и работает с сервисами.
